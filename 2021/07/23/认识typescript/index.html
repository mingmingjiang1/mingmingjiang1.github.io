认识Typescript

一、Typescript和JavaScript

  1. 大部分编程语言编写的程序都会经过下面的几个步骤转换成机器语言

    文件中的文本->编译器->AST->字节码

    但是Typescript它并不是直接转换成字节码的，而是转换成Javascript的；并且在转换过程中包含类型检查，整个TS解析成机器语言的过程如下：

    TS部分：

    • TS源码 -> TS AST
    • 类型检查AST
    • TS AST -> JS源码

    JS部分：

    • JS源码 -> JS AST
    • AST -> 字节码
    • 运行时计算字节码

    TS部分右TS compiler操作，而JS部分由node解释器或者浏览器操作

  1. TS有一个很重要的功能: 提前类型检查

    一般类似类型检查由两种：显示声明和自动推导

    TS两者皆有

  2. JS vs TS

类型系统特性 JS TS
类型绑定 动态(运行时确定) 静态(编译时确定)
是否自动类型转换 是 否(多数时候)
何时进行类型检查 运行时 编译时
何时报错 运行时(多时) 运行时(多时)

动态与静态:

JS是动态绑定的，只有运行程序的时候才能知道类型；相反TS是静态绑定，在编译的时候确定类型;也正因为这个原因，JS是弱类型语言，它默认允许很多的隐式的类型转换，而TS却不允许，它在编译的时候，就及时发现这些错误，并捕获，在编辑器里书写代码的时候，就捕获这些错误，但有些错误也是TS无法提前捕获的

**Tips:**动态与静态，强类型和弱类型
