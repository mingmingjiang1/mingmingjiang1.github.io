跨域（二）

Created2021-07-22|Updated2021-07-22|前端
|Post View:

Cross-origin requests( CORS) 跨域资源共享是一种机制，该机制使用附加的HTPP头部告诉浏览器，准许运行一个在源上的Web应用访问位与另一不同源选定的资源。当一个Web应用发起一个与自身所在源(协议, 域，端口)不同的HTTP请求，它发起的即跨源HTTP请求。

但是处于安全性，浏览器限制脚本内发起的跨源HTTP请求。如XMLHttpRequest和Fetch API都遵循API同源策略。这就意味着使用这些API的Web应用程序只能从加载应用程序的同一个域请求HTTP资源，除非响应报文设置了某些CORS响应头。

如何通过设置HTTP头部来CROS呢？（以下的设置仅限于http1.0）

跨源资源共享标准新增了一组 HTTP 首部字段，允许服务器声明哪些源站通过浏览器有权限访问哪些资源。另外，规范要求，对那些可能对服务器数据产生副作用的 HTTP 请求方法（特别是 GET 以外的 HTTP 请求，或者搭配某些 MIME 类型的 POST 请求），浏览器必须首先使用 OPTIONS 方法发起一个预检请求（preflight request），从而获知服务端是否允许该跨源请求。服务器确认允许之后，才发起实际的 HTTP 请求。在预检请求的返回中，服务器端也可以通知客户端，是否需要携带身份凭证（包括 Cookies 和 HTTP 认证相关数据）

常见的使用http头部设置来进行CROS的场景：

  1. 简单请求，methods是GET,POST,HEAD这样的方法，而content-type是以下三个类型之一：

    • texte/plain
    • multipart/form-data
    • application/x-www-form-urlencoded

    出来上述的要求，还有其他的一些要求，这里不再叙述

    对于简单请求，如果希望跨域访问，可以在响应头中设置：

    Access-Control-Allow-Origin: http://foo.example

    表明该响应头仅仅针对http://foo.example发出的请求才做出回应，所以在请求头的origin必须要设置：http://foo.example，类似的

    Access-Control-Allow-Origin: *

    它可以对任何请求源做出跨域响应

  2. 附带cookie的请求

    一般而言，对于跨源 XMLHttpRequest 或 Fetch 请求，浏览器不会发送身份凭证信息。如果要发送凭证信息，需要设置 XMLHttpRequest 的某个特殊标志位。

    与简单请求不同的是，必须在响应头中额外附加一个参数：

    Access-Control-Allow-Credentials: true

    同时，请求头中也必须有一个表明携带了身份验证的参数

    withCredentials = true，

  3. 预检请求

    与前述简单请求不同，“需预检的请求”要求必须首先使用 OPTIONS 方法发起一个预检请求到服务器，以获知服务器是否允许该实际请求。”预检请求“的使用，可以避免跨域请求对服务器的用户数据产生未预期的影响，预检请求可以设置一个参数Access-Control-Allow-Age:表明当前预检请求最大的生命时间，在这段时间内，发送实际请求，不会再次发送预检请求。

在vue的使用过程中，我们一般都是使用vue的开发服务器运行程序，当我们在浏览器中请求该服务器时，相当于是从客户端->localhost服务器的一个请求，这是允许的；但是如果直接通过浏览器去请求跨域的数据，这是不可取的，这个时候必须把localhost当作中间的一个代理，把所有有关跨域的请求都委托给代理，比如：我想从服务器获取http:bilibili/api/list的数据，但是直接使用该api是跨域的，这个时候可以委托给代理，同时不要直接发送该请求获取数据：/api/list,然后在配置文件中设置代理：

proxy: {

​ “/api”: target: “http:bilibili/api/list”

}

这样在浏览器发送的请求就不含跨域名的了，而且会委托给代理发送跨域请求，但是这样有个缺点，只能在开发模式下，使用该方法，因为配置文件本身并不会打包到最终的文件。那为什么代理可以跨域呢？

网友1的回答：同源策略是浏览器的安全策略，不是HTTP协议的一部分。

服务器端调用HTTP接口只是使用HTTP协议，不会执行JS脚本，不需要同源策略，也就没有跨越问题；

网友2的回答：为什么会有跨域，那是浏览器的安全策略，没了浏览器就没有跨域了
